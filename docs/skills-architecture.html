🔍 shouldIncludeSkill() 判断逻辑
✅ 通过条件
• enabled !== false (config)
• isBundledSkillAllowed (allowBundled 白名单)
• OS 平台匹配 (metadata.os)
• 所需环境变量存在 (requires.env)
• 所需二进制工具可用 (requires.bins)
• 所需配置路径有效 (requires.config)
• always: true 则跳过环境检查
❌ 排除条件
• enabled: false (显式禁用)
• 不在 allowBundled 白名单
• OS 不匹配
• 缺少必需环境变量
• 缺少必需二进制工具
• 文件超过 maxSkillFileBytes
• 路径逃出配置的根目录
🤖 模型侧调用流程（Prompt 驱动路径）
1
会话开始 — 构建 SkillSnapshot
ensureSkillSnapshot() 在首回合或版本刷新时调用，将技能快照写入 sessionEntry.skillsSnapshot
// session-updates.ts ensureSkillSnapshot(session, workspaceDir, config) → buildWorkspaceSkillSnapshot(workspaceDir, opts) → { prompt, skills[], version }
2
构建 System Prompt — 注入 Skills 段落
resolveSkillsPromptForRun() 从快照取 prompt，buildSystemPrompt() 将其嵌入 ## Skills (mandatory) 段落
// system-prompt.ts → buildSkillsSection() "## Skills (mandatory)" "Before replying: scan <available_skills> <description> entries." "- If exactly one skill clearly applies: read its SKILL.md at <location> with `read`, then follow it." "- If multiple could apply: choose the most specific one, then read/follow it." "- If none clearly apply: do not read any SKILL.md." "Constraints: never read more than one skill up front; only read after selecting." // ... + skillsPrompt (available_skills 列表)
3
模型收到对话 — 扫描 available_skills
模型读取 system prompt 中的 <available_skills> 块，扫描每个技能的 name、description、location 字段
<available_skills> <skill name="multi-search-engine" description="17 search engines for web crawling..." location="~/skills/multi-search-engine/SKILL.md" /> <skill name="github" ... /> ... </available_skills>
4
选择技能 — 调用 Read 工具读取 SKILL.md
模型决定使用某个技能后，用 Read 工具（文件工具）读取对应的 SKILL.md 文件内容
// 模型发出 tool call read({ path: "~/skills/multi-search-engine/SKILL.md" }) // → 返回 SKILL.md 全文内容
5
执行技能指令
模型按 SKILL.md 中的指南执行任务（如调用 web_fetch 搜索、调用 GitHub API 等）
// 按 SKILL.md 内容执行，例如 multi-search-engine： web_fetch({ url: "https://www.google.com/search?q=python+tutorial" })
⌨️ 斜杠命令调用路径（/命令名）
路径 A：dispatch → Tool 直接派发
用户输入 /search python
resolveSkillCommandInvocation() 匹配技能名
读取 frontmatter 中 command-dispatch: tool
找到 toolName 对应的 AnyAgentTool
tool.execute(args) 直接调用，绕过 LLM
// SKILL.md frontmatter command-dispatch: tool command-tool: web_search command-arg-mode: raw
路径 B：改写消息 → 走 Agent 流程
用户输入 /github create issue
resolveSkillCommandInvocation() 匹配技能名
无 dispatch 配置
消息体改写为「请使用 github skill...」
进入正常 agent 对话，模型自行读取 SKILL.md
// SKILL.md frontmatter user-invocable: true // 无 command-dispatch
📄 SKILL.md 文件结构
--- name: my-skill # 技能唯一名称（必须） description: 用一句话描述技能功能 # 出现在 available_skills 列表（必须） user-invocable: true # 是否可用 /命令名 调用（默认 true） disable-model-invocation: false # 禁止模型主动调用此技能（默认 false） command-dispatch: tool # 斜杠命令直接派发到 tool（可选） command-tool: web_search # 目标 tool 名称 command-arg-mode: raw # 参数传递模式 metadata: openclaw: always: false # true = 始终包含，跳过环境检查 skillKey: my-skill-key # 配置文件中的 key（默认等于 name） primaryEnv: MY_API_KEY # 主要环境变量（apiKey 配置时自动注入） emoji: "🔍" homepage: https://example.com os: [darwin, linux] # 仅在特定 OS 上启用 requires: bins: [ffmpeg] # 必须存在的可执行文件 anyBins: [brew, apt] # 任一存在即可 env: [MY_API_KEY] # 必须存在的环境变量 config: [browser.enabled] # 必须为 truthy 的配置路径 tools: [web_fetch] # 依赖的 agent 工具 install: - kind: brew formula: ffmpeg - kind: node package: @my/tool --- # 技能正文（Markdown 格式，模型执行时读取） ## 使用场景 当用户需要 ... 时，使用本技能。 ## 操作步骤 1. 首先检查 ... 2. 然后调用 web_fetch({url: "..."}) 3. 处理返回结果 ... ## 注意事项 - 注意 rate limit：...

🔄 快照版本刷新机制
1
refresh.ts — 监听 SKILL.md 变化
文件系统 watcher 监听 skills 目录下的 SKILL.md 文件，变化时递增 snapshotVersion
2
ensureSkillSnapshot() — 版本比对
每次 agent 回合开始时，比对 session.skillsSnapshot.version 与当前版本号，不一致则重建快照
3
serialize.ts — 串行化锁
serializeByKey() 确保同一 workspaceDir 的技能构建不并发执行，防止竞态条件
